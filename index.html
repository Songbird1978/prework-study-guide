HTML


  • HTML is the code language that means Hypertext Markup Language.

  • It lays down the structure of a webpage.

  • It contains elements such as:

  • Head
  • Meta
  • Body
  • Lists
  • Style
  • Base
  • Script

  • The most basic parts of a page structure are:

  • Head - The head element contains information about the webpage. This includes elements such as metadata (main language, performance, search criteria, viewing sizes (viewport), titles and character sets) and will usually not be seen by the audience.

  • Body - The body element contains the information such as pictures, links, lists, paragraphs, text and headings inside sections

  • Section - This is a way to group specific parts of the body of the page for further processing later.

  • Lists - Are an element in which bullet points or numbered lists can be ordered or unordered.

  • Index.html is always the homepage of a site where the HTML is stored and links to other types of code files for the site such as CSS and Javascript.

  • We can use VS Code or other software to help us code and use the language and tags.


CSS


  • CSS means Cascading style sheets

  • They add styling information to your HTML code to improve the aesthetics of a page.

  • There are 3 types:

  • In-line CSS = adds directly to a specific area of the HTML (eg. A heading)

  • Internal style sheet = CSS code embedded within the webpage (normally the [head])

  • External style sheets = CSS code created in an external document but added as a [link] within the [head]) - this makes any changes easier to apply.

  • Separation of concerns is a term used to refer to the practice of keeping code responsibilities (design, features, procedures, style, functions) separate. In audio engineering terms, it is "good housekeeping"

  • External css documents are normally labelled style.css

  • Creation of CSS external sheet includes, margin & padding.

  • Margin = space around the outside of the element - buffer (invisible)

  • Padding = space around the content inside the element

  • Borders = thickness of the border around the box

  • CSS boxes can be styled:

  • Inline (principles applied on the same line) or;

  • block (principles applied on a new line)

  • DRY = do not repeat yourself = re-use or share pieces of code to simplify & reduce size of code

  • Use comma to group elements together that will be styled the same way


GIT


  • Git is a language used by GitHub to communicate between the shell and the cloud. It allows you to perform specific tasks related to your repositories or repos, which are basically projects.

  • There are commands based on:-

  • Setup = for configuring user information across platforms

  • Setup & Initialization = to clone repositories and choose directories to work on

  • Stage and Snapshot = to save versions of your work, comment and commit for review

  • Branch & Merge = to create new branches, check work done and merge with main directories

  • Inspect & Compare = inspecting what work you have done in comparison


JAVASCRIPT


  • Javascript adds interactivity to a website.

  • Javascript file extension = .js

  • Variables (var =) are containers that store values

  • A semicolon at the end of the line is where a statement ends;

  • The javascript file is included at the end of the body part of the HTML file.

  • Javascript needs to be read AFTER the HTML coding is established.

  • In order to use JS - you must understand control flow.

  • Control flow is where actions will be performed in order, unless certain conditions are met. Truthy or Falsey answers will lead to the next action (question) until the actions are back on the same pathway.

  • A good way to understand control flow is to imagine a motorway with traffic. A car on the motorway will continue to drive forward on the motorway unless certain conditions are met, which require it to leave and then return to the flow. Those conditions could be leaving for food, toilet or petrol. IF those elements have been identified and then fulfilled, the car will return to the flow of the traffic on it's journey.

  • Using the IF + statement, to propose different actions IF certain statements are truthy, then an action will occur.

  • Strings represent textual data. Strings cannot be modified.

  • Strings need to be contained within quotation marks.

  • An array is a function which enables more than one type of statement or piece of information to be stored ready for selection and further action.

  • To work with individual data within the array, we need to add a location by way of an index.

  • We can use a for loop to operate on each element in an array.

  • We can add functions using the function keyword

  • Functions are reusable blocks of code that contain instructions on how to perform a task.

  • Functions need to be named and be followed by parentheses.

  • We add curly brackets to contain the code for the task.

  • We can check our code is working using the console tab in a browser window.

  • Math.floor() and Math.random() can be used together to generate a random number.

  • To execute a function, we must call it in our code

  • ORDER MATTERS! - Javascript runs from top to bottom!
